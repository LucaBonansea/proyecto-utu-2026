# Entrega 1

## Idea del proyecto

El proyecto consiste en el desarrollo de un sistema informático orientado a resolver una problemática real o simulada, aplicando los conceptos de Ingeniería de Software.

En esta primera etapa, el enfoque estará en comprender las necesidades del cliente y definir una base clara para el desarrollo del sistema.

---

## Requisitos iniciales

### Recolección de información (cliente)

Se realizará una instancia de relevamiento mediante preguntas al cliente con el objetivo de comprender el problema y definir los requerimientos del sistema.

#### Ejemplo de preguntas:

* ¿Qué problema desea resolver?
* ¿Quiénes serán los usuarios del sistema?
* ¿Qué tareas desea automatizar?
* ¿Qué información necesita gestionar?
* ¿Existen sistemas actuales? ¿Qué limitaciones tienen?
* ¿Qué funcionalidades considera más importantes?
* ¿Desde qué dispositivos se utilizará el sistema?

---

### Requerimientos funcionales (iniciales)

A partir de la información obtenida, se definirán los primeros requerimientos, tales como:

* Registro e inicio de sesión de usuarios
* Gestión básica de información (crear, editar, eliminar)
* Visualización de datos
* Diferenciación de roles (usuario / administrador)

---

### Requerimientos no funcionales

* Facilidad de uso (interfaz intuitiva)
* Seguridad en el manejo de datos
* Buen rendimiento
* Accesibilidad

---

## Análisis básico

### Problema a resolver

Se busca identificar y analizar una problemática concreta del cliente, relacionada con la gestión de información o procesos que actualmente se realizan de forma manual o ineficiente.

---

### Objetivo del sistema

Desarrollar una solución que permita mejorar la organización, automatizar tareas y facilitar el acceso a la información.

---

### Actores del sistema

* Usuario
* Administrador

---

## Diseño inicial (Frontend)

En esta etapa se comenzará con el diseño de las primeras interfaces del sistema (frontend), con el objetivo de definir la interacción del usuario.

Se desarrollarán prototipos básicos de:

* Pantalla de inicio
* Pantalla de login
* Pantalla principal del sistema
* Formularios básicos

Estos prototipos permitirán validar ideas con el cliente antes de avanzar al desarrollo completo.

---

## Metodología de trabajo

El equipo trabajará utilizando:

* Repositorio en GitHub
* Gestión de tareas mediante Issues
* Tablero Kanban para organización
* Uso de ramas (feature) para desarrollo
* Pull Requests para integración de cambios
